## Incident Log - что пошло не так и как решалось

### 2026-06-08 — MTProto прокси не работает
- **Что делал**: настроил MTProto прокси, используя nginx в качестве маск-хоста.
- **Результат**: прокси не заработал.
- **Диагностика**: проверка логов, тесты с разных IP, тестирование готовых серверных проектов с подобной концепцией.
- **Вывод**: MTProto полностью блокируется по сигнатуре в сети.
- **Решение**: проект отложен, эксперимент зафиксирован. Использование локальных решений на основе MTProto для каждого устройства по отдельности.

### 2026-06-13 — отключение света
- **Проблема**: после восстановления питания контейнеры не запустились автоматически.
- **Решение**: добавил `restart: unless-stopped` для всех сервисов. Теперь docker запускает их после загрузки системы.
- **Время восстановления**: примерно 5 минут.
